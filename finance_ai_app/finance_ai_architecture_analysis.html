Finance AI app — full architecture and technical analysis

App architecture overview

Finance AI is a desktop GUI application built with PyQt6. It follows a classic Layered + Event-Driven hybrid architecture, decomposed into four horizontal layers that communicate primarily through a central EventBus (observer/pub-sub pattern) rather than direct coupling.

AI subsystem deep-dive

Plugin architecture — Strategy + Registry

All three LLM backends implement BaseAIPlugin (abstract class). The registry holds a dict of engine_id → plugin. ChatbotFrame calls registry.get(engine_id).create_worker(...) — it never imports a specific worker class directly.

ChatbotFrame → registry.get(id) → BaseAIPlugin.create_worker() → QThread worker → token_received signal → UI bubble.append_text()

Three inference engines

Gemini (cloud)
GeminiWorker
SSE streaming via urllib (no extra SDK). Converts chat history to Gemini contents format. Handles 429 rate-limits and 400 bad-request distinctly.
Ollama (local)
OllamaWorker
HTTP POST to localhost:11434/api/chat with streaming JSON lines. Auto-selects first available non-cloud model from the Ollama tag list.
Embedded (offline)
EmbeddedLLMWorker
HuggingFace AutoModelForCausalLM + TextIteratorStreamer. Uses fine-tuned DistilGPT2 if available, else downloads TinyLlama-1.1B-Chat. GPU auto-detected via torch.cuda.

ML pipelines

classifier
TF-IDF (char n-grams 2–4) + RandomForest, 200 estimators
Single sklearn Pipeline. Seeds from 48 hard-coded labelled pairs; extends with every user transaction that has a category. Singleton with double-checked locking; persisted as pickle. Retrains on demand when user corrects a category.
forecaster
Prophet (preferred) / 3-month moving average (fallback)
Selects Prophet when ≥3 months of data exist and the library is installed. Prophet configured with monthly seasonality disabled; changepoint_prior_scale=0.3; 80% confidence interval. MA uses numpy mean + std.
anomaly
Isolation Forest (sklearn), contamination=0.08
5 engineered features: log-amount, per-category Z-score, global amount ratio, binary late-hour flag, day-of-week. StandardScaler normalisation before fitting. 200 trees, n_jobs=-1. score_samples() output converted to risk_score (0–99). Results written back to transactions.is_anomaly.
fine-tuner
DistilGPT2 fine-tune via HuggingFace Trainer
Generates Q&A training pairs from actual DB data (monthly summaries, category totals, plus static financial-literacy examples). Causal LM objective, max_length=256, 3 epochs, fp16 on GPU. Saves model + tokenizer locally for EmbeddedLLMWorker to load.

Core algorithms & CS concepts

pub-sub
Event Bus — Observer pattern via PyQt signals
_EventBus(QObject) is a singleton with 20+ typed signals. Frames subscribe via bus.signal.connect(slot). Debounce implemented with QTimer dict to coalesce burst events (e.g. bulk CSV imports).
concurrency
Thread isolation for all I/O-bound and CPU-bound work
Every AI inference (LLM, forecast, anomaly, classifier) runs in a QThread subclass. Results are communicated back via pyqtSignal — never touching Qt widgets from the background thread. DatabaseManager uses a threading.Lock for write serialisation on the shared SQLite connection.
data structure
Context manager for DB transactions
DBContext.__enter__/__exit__ wraps every SQL block with commit-on-success / rollback-on-exception. Prevents partial writes. A generator variant db_session() provides the same guarantee via contextlib.contextmanager.
search
Scored fuzzy search in command palette
Command.score(query) returns 100 (prefix match), 50 (substring in title), 10 (keyword match). Results are sorted descending, capped at 12 items. Runs synchronously on keypress — fast enough for <50 commands.
regex/NLP
Regex-based amount normalisation
nlp_parser.py uses a single compiled regex (\d+(?:[.,]\d+)?)\s*(k|nghìn|triệu|tr|đ|đồng)? to capture amounts and unit suffixes. Handles Vietnamese decimal/thousands ambiguity (comma vs dot), then multiplies by 1,000 or 1,000,000 as needed.
parsing
Multi-profile CSV column detection
CsvImporter._map_columns() performs fuzzy column header matching: for each profile's candidate list, tries cand in col or col in cand — handles partial/translated headers. Bank auto-detection checks filename then first-500-char content for bank-specific keywords.
singleton
Thread-safe Singleton for DatabaseManager and TransactionClassifier
Both use the __new__ override with a class-level threading.Lock for double-checked locking. DatabaseManager also probes with SELECT 1 to detect stale connections and reconnects automatically.
lazy loading
On-demand page instantiation
MainWindow._navigate(page) checks self._pages dict before creating a frame. First visit constructs and adds to QStackedWidget; subsequent visits just switch the active widget. Avoids constructing all 8 frames at startup.

Design patterns applied

Strategy
AI plugin system
BaseAIPlugin defines create_worker() and check_status(). Three concrete strategies: GeminiPlugin, OllamaPlugin, EmbeddedPlugin. Swapped at runtime via combobox in the chat UI — no conditional branches in the calling code.
Observer
Event Bus (PyQt signals)
Completely decoupled pub-sub. DashboardFrame never imports TransactionFrame; it just subscribes to bus.transaction_added. BusConnectMixin provides lifecycle-safe connect/disconnect hooks.
Repository
Data access abstraction
TransactionRepo, BudgetRepo, CategoryRepo, AccountRepo, UserProfileRepo encapsulate all SQL. Domain logic never writes raw queries. Repo methods validate input via DataModel.validate() before executing.
Facade
TransactionManager
UI frames call self.tm.add_transaction() — the manager internally calls the repo, updates the account balance, checks budgets, and emits bus events. Simplifies the UI layer: one method call triggers the full domain chain.
Singleton
DatabaseManager, TransactionClassifier, EventBus, Notifier, ThemeEngine, AIPluginRegistry
All 6 are module-level singletons instantiated once at import time (or via __new__). Ensures one DB connection, one classifier model in memory, one event hub.
Factory
MainWindow._create_page()
Centralised factory method maps page names to frame classes. Keeps _navigate() clean and makes adding new pages a one-line change.
Template Method
ReportGenerator sections
generate_monthly_report() defines a fixed sequence of build steps (_build_header → _build_summary_cards → _build_category_section → ...). Each step is a separate overridable method returning a ReportLab flowable list.
Proxy
DatabaseManager.__getattr__
Forwards unknown attribute access to the underlying sqlite3.Connection, so callers can use conn.execute() directly without an explicit getter. Transparent wrapper.

Connection strategy

PRAGMA journal_mode = WAL allows concurrent reads while writing. PRAGMA foreign_keys = ON. Write operations serialised with a class-level threading.Lock. Two access paths coexist: the legacy DatabaseManager singleton (shared connection) and the newer db_session() generator (per-call connection, cleaner isolation) — both are used in the codebase.
